# Individual Contributions

This section documents how the work was divided across four team members. The names below are temporary placeholders and can be replaced with the final student names later.

:::info Contribution Structure
The project was split into four balanced tracks: research and requirements, interaction design, frontend implementation, and technical architecture/evaluation. Each member owned one major track while reviewing and supporting the others.
:::

## Contribution Overview

| Member | Main Responsibility | Core Output |
| --- | --- | --- |
| Student A | Research, requirements, and UX strategy | Stakeholder analysis, user journey map, requirements list, social/ethical reflection |
| Student B | Interaction design and visual prototyping | Crazy 8s rationale, design alternatives, map/wall/collection/profile interaction decisions |
| Student C | Frontend implementation and product logic | Reusable MDX components, responsive UI layouts, documentation integration, build checks |
| Student D | 3D/AR pipeline, system architecture, and evaluation | Architecture documentation, AR performance analysis, iteration log, future roadmap |

## Student A - Research, Requirements, and UX Strategy

**Primary ownership**

- Defined the target users and stakeholder groups for campus exploration.
- Mapped the journey from orientation anxiety to playful discovery.
- Translated research findings into three core product requirements: NFC check-in and collection, AR companion photo op, and location-based message wall.
- Maintained the project narrative across motivation, requirements, and final reflection.

**Key deliverables**

| Deliverable | Contribution |
| --- | --- |
| Stakeholder analysis | Identified the needs of visitors, freshmen, and returning students. |
| User journey map | Compared the current frustrating journey with the improved Campass experience. |
| Requirements list | Converted research insights into measurable product requirements. |
| Reflection | Discussed social value, privacy, accessibility, and responsible AI use. |

## Student B - Interaction Design and Visual Prototyping

**Primary ownership**

- Explored Crazy 8s alternatives for Map, Wall, Collection, and Profile flows.
- Compared map paradigms and selected the custom narrative-stage campus map direction.
- Designed responsive interaction patterns such as mobile bottom drawers and desktop master-detail side panels.
- Kept the interface consistent with the Campass goal: playful, clear, and easy to scan.

**Key deliverables**

| Deliverable | Contribution |
| --- | --- |
| Crazy 8s | Documented selected and discarded ideas for core interaction modules. |
| Design alternatives | Compared map and map-pin interaction approaches with design rationale. |
| Low-fi prototype | Helped shape the early structure before high-fidelity implementation. |
| Interaction decisions | Connected user needs with practical screen-level layout choices. |

## Student C - Frontend Implementation and Product Logic

**Primary ownership**

- Implemented React/Docusaurus documentation components used across the project site.
- Built reusable MDX panels for architecture, refinement, reflection, and visual explanation pages.
- Integrated theme-aware styling, responsive layouts, Lucide icons, and interactive UI patterns.
- Ran build and type checks to keep the project deployable through GitHub Pages.

**Key deliverables**

| Deliverable | Contribution |
| --- | --- |
| Component library | Created reusable visual sections for project documentation. |
| Responsive UI | Ensured pages adapt across desktop and mobile screen widths. |
| MDX integration | Connected written documentation with interactive React components. |
| Build maintenance | Checked Docusaurus build output and fixed implementation issues. |

## Student D - 3D/AR Pipeline, Architecture, and Evaluation

**Primary ownership**

- Designed the prototype architecture around static hosting, LocalStorage, and client-side interaction.
- Investigated the AR model assembly pipeline and documented performance limitations.
- Evaluated failed approaches such as DOM slot nesting, serverless compression, and client-side Draco compression.
- Planned the production roadmap toward Next.js, persistent storage, and scalable asset delivery.

**Key deliverables**

| Deliverable | Contribution |
| --- | --- |
| System architecture | Explained the prototype architecture and production migration direction. |
| AR performance analysis | Documented the mobile performance bottleneck caused by 3D asset assembly. |
| Iteration log | Recorded technical attempts, failures, and pivots. |
| Future roadmap | Proposed the next step toward server-side optimization and scalable deployment. |
