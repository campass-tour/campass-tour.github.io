# Requirements List

To be considered truly **Playful**, the system must satisfy all three requirements below. Each requirement is defined not only by feature presence, but also by whether it creates delight, agency, and social momentum during campus exploration.

      <li style={{ marginBottom: '1rem' }}>
        <strong>NFC Tangible Check-in & Collection:</strong> Users must be able to physically tap NFC tags at landmark spots to complete check-ins, unlock spot-specific micro-stories, and collect location-themed digital items. This requirement turns passive walking into an active treasure-hunt loop, where each visit produces visible progress and a sense of achievement. A core acceptance signal is that users can clearly track what they have collected and what remains to be explored.
      </li>
      <li style={{ marginBottom: '1rem' }}>
        <strong>AR Companion Photo Op:</strong> After unlocking companions, users must be able to summon them in AR for photo-taking and sharing in real campus settings. This requirement makes the experience expressive rather than purely functional, allowing users to create personal memories and socially shareable moments tied to specific places. A core acceptance signal is that users can complete the AR photo flow quickly without technical friction.
      </li>
      <li>
        <strong>Location-based Message Wall:</strong> Each check-in point must have a geo-locked message wall where on-site users can read and leave place-specific notes, tips, and media. This requirement extends playfulness into community continuity, so each location feels "alive" with contributions from previous and current students. A core acceptance signal is that users can both consume and contribute meaningful localized content after successful check-in.
      </li>
    </ul>
  </div>
</div>
